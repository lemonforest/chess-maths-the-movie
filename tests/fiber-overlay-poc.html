PATH A · per-square background-image

64 discrete colors, one per square. Same mechanism as the existing channel overlay (js/chess-overlay.js). Honest "value at each square". Pieces render on top via normal CSS stacking — no z-index tweaks needed because background-image is below the piece <img>.

PATH B · smooth canvas overlay

Upsampled bilinear canvas laid over the board with CSS absolute positioning; alpha blends with the squares. Pieces raised to z-index 4 so they stay legible. Board flips and resizes re-run layoutGradient().

PATH C · separate SVG overlay

Independent SVG document positioned over the board. Same alignment concerns as Path B. Useful if we want SVG filters (e.g. Gaussian blur) rather than raster upsampling. For this POC we draw 64 rects with opacity — a SMIL-free, Tiny-1.2-safe subset — which degenerates into "Path A with extra steps" unless we add <feGaussianBlur>.
